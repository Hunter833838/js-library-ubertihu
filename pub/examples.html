Initialization

To convert this table to a BetterTable, run the line of code below:

     // The element ID of this table is "standardTable"
        let myBetterTable = new BetterTable("standardTable");

This is a BetterTable! We haven't added any features yet, so basically we've just applied the custom BetterTable Styling.

Documentation explains how to override this styling

There are multiple other ways to instantiate a BetterTable, see the code blocks below:

     // PASSING A NODE ELEMENT
        let table = document.getElementById("standardTable");
        let myBetterTable = new BetterTable(table); 
     // POPULATE IT YOURSELF
        let headers = ["#", "Team", "Pl", "W", "D", "L", "F", "A", "GD", "Pts"]
        let rows = [[1, "Manchester City", 30, 22, 5, 3, 64, 21, 43, 71],
                [2, "Manchester United"...]...]
        let args = {tableData: {headers: headers, rows: rows}}
        let myBetterTable = new BetterTable(table, args);

What's that args parameter? Don't worry, we'll touch on it later...

Suppose we wanted to add an additional (imaginary) team to the Premier League table, we would run this block of code:

Note

The examples showed how to individually enable each of the features after initial BetterTable instantiation

But it's also possible to enable all of these features during instantiation through the args parameter

       let args = {sortable: true, 
          exportToCSV: true, 
          reorderColumns: true, 
          search : {headerToSearchBy : "Team"}}
        let myBetterTable = new BetterTable("standardTable", args)

The BetterTable constructor will just all of the above methods for you, as you specify them in the args parameter!
